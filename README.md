# Frontend 깃 컨벤션

- feat : 새로운 기능 추가
- add : 새로운 코드,파일 생성 
- delete : 사용하지 않는 코드,파일 삭제 
- chore : 간단한 코드 수정 (변수명 등)
- ui : ui 작업
- docs : 문서 수정 (readme)
- refactor : 코드 리펙토링
- fix : 버그 수정

# Frontend 브랜치 컨벤션
- 1. 이슈 파기 -> 작업 내용
  3. 브랜치 파기 -> #번호/작업 내용 (ex- #1/로그인)
  4. 작업하기 -> 커밋 메세지 : 유형 #번호/작업 내용 (ex- ui #1/로그인 컴포넌트 생성)
